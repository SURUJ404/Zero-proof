# tsc-ai

AI-driven evasion engine — fingerprints your environment, detects sandboxes/AV/EDR, recommends optimal evasion techniques, and adapts beacon timing via reinforcement learning.

## Installation

```bash
npm install -g tsc-ai
```

## Quick Start

### Full environment scan

```bash
tsc-ai scan
```

### Scan + evasion recommendation

```bash
tsc-ai evade
```

### Sandbox/VM detection

```bash
tsc-ai sandbox-test
```

### Adaptive beacon (simulated)

```bash
tsc-ai beacon --interval auto
```

### Train the RL model

```bash
tsc-ai learn --train
```

### Full JSON report

```bash
tsc-ai report --json
```

## Evasion Decision Tree

```
┌─ Is sandbox? ──→ YES → Go dormant (no C2, pretend benign)
│
└─ Is EDR present? ──→ YES → Direct syscalls (Hell's Gate/Halo's Gate)
│                            + ETW patching + AMSI bypass
│
└─ Is AV present? ──→ YES → Technique depends on AV:
│    ├─ Defender → AMSI bypass + ETW patch
│    ├─ Kaspersky → Unhook ntdll + direct syscalls
│    └─ Other → Process hollowing + PPID spoof
│
└─ Is admin? ──→ YES → Kernel callbacks + driver loading
│
└─ No AV/admin → Plain shellcode injection
```

## Beacon Q-Learning

The adaptive beacon engine uses Q-learning:
- **States**: environment type (normal/sandbox/AV-detected) + network type (NAT/Direct/Proxy) + hour
- **Actions**: beacon intervals (1s, 5s, 30s, 5min, 30min, 1hr) + jitter (0-100%)
- **Reward**: successful beacon → higher interval + lower jitter
- **Penalty**: failed beacon → lower interval + higher jitter + channel switch
